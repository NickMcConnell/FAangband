# File: k_info.txt


# This file is used to initialize the "lib/raw/k_info.raw" file, which is
# used to initialize the "object kind" information for the Angband game.

# Do not modify this file unless you know exactly what you are doing, 
# unless you wish to risk possible system crashes and broken savefiles.

# In Oangband, objects may have descriptions.  These descriptions only appear 
# if the player is at least aware of the object kind.  Food, weapons, armours, 
# junk, and tools have complete descriptions, while those of magical devices 
# also use the object flavor (the kind of material) and often insert 
# extra numerical information (indicated by a '~').  Potions, scrolls, chests, 
# and spellbooks are not described.  Artifacts are described in a_info.txt.

# Special thanks to J.R.R Tolkien for fundamental concepts and a poem, and 
# to Pat Tracy, a primary source for weapon and armour descriptions.

# Note that object zero is used for the "stack" picture (unused).


# Version stamp (required)

V:1.1.6

#N: object index: name
#G: ASCII char used to display the object: color symbol
#I: object tval (general kind): object sval (specific type): 
#     pval (controls bonuses to food value, stats, charges, and skills)
#W: where the object begins to be found in-depth: unused: 
#     weight (in deci-pounds): value
#P: Base armour class: damage done when wielded or thrown: Bonus to Skill: 
#   Bonus to Deadliness: Bonus to armour class: Activation index
#A: All depths/rarities of the item (max of four pairs possible)
#E: effect when used : recharge time (if applicable)
#F: All item-specific flags (special qualities).  Some depend on pval.
#D: The object descripion, used to provide more information when it is 'I'nspected.

# These objects do not specify "full names" because the artifact name
# is added in "obj-desc.c" based on the artifact index ("i_ptr->name1").
#

# is "aware" (always true for lites)
#
# The Lites (and the One Ring) specify "physical colors", which
# over-ride the "flavor" colors, if any.  Note that the "One Ring"
# also has a specific check for "unknown" in which it changes its
# name to "a plain gold ring".  See "obj-desc.c"
#

# so we do not need to specify that here.
#
# Note that the "INSTA_ART" flag is used to prevent these objects from
# being created without also turning them into artifacts.  This flag
# must be specified both here and in the artifact template.
#
# The only reason for having six different "ring" templates and five
# different "amulet" templates is to allow each "special artifact" to
# have a different "color" and "flavor", and also to allow the use of
# special "base names" (such as "Necklace").
